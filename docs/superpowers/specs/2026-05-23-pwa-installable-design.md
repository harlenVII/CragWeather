# PWA Installable — Design Spec

**Date:** 2026-05-23  
**Scope:** Make CragWeather installable as a PWA (Add to Home Screen). No offline support, no caching.

---

## Goal

Users on mobile and desktop should be able to install CragWeather as a standalone app. The browser install prompt should appear, and the app should open without browser chrome when launched from the home screen or dock.

## Approach

Manual implementation — no new runtime dependencies, no Workbox. A minimal pass-through service worker satisfies Chrome's installability requirement without changing any network behavior.

---

## Files to Create

### `public/manifest.json`

```json
{
  "name": "CragWeather",
  "short_name": "CragWeather",
  "start_url": "/",
  "display": "standalone",
  "theme_color": "#c2410c",
  "background_color": "#fafafa",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

### `public/sw.js`

A no-op service worker — satisfies Chrome's SW requirement for the install prompt, does zero caching:

```js
self.addEventListener('fetch', (event) => {
  event.respondWith(fetch(event.request));
});
```

### `public/icon-192.png` and `public/icon-512.png`

Placeholder icons: `#c2410c` (accent) background, white "CW" text centered. Generated by `scripts/generate-icons.ts` (one-time run).

### `scripts/generate-icons.ts`

One-time script using the `canvas` devDependency to write both PNGs into `public/`. After running, this script and its dependency can remain as devDependencies (no impact on production build).

### `components/ServiceWorkerRegistration.tsx`

Client component that registers the SW on mount:

```tsx
"use client";
import { useEffect } from "react";

export default function ServiceWorkerRegistration() {
  useEffect(() => {
    if ("serviceWorker" in navigator) {
      navigator.serviceWorker.register("/sw.js");
    }
  }, []);
  return null;
}
```

## Changes to Existing Files

### `app/layout.tsx`

Add to `<head>`:

```html
<link rel="manifest" href="/manifest.json" />
<meta name="theme-color" content="#c2410c" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="default" />
<meta name="apple-mobile-web-app-title" content="CragWeather" />
<link rel="apple-touch-icon" href="/icon-192.png" />
```

Add to end of `<body>`:

```html
<ServiceWorkerRegistration />
```

---

## What This Does NOT Do

- No caching of any kind — all network requests pass through unchanged
- No offline support — app requires network for all weather and route data
- No background sync, push notifications, or other PWA features
- `/api/*` routes are unaffected

## Dependencies

- **`canvas`** — added as devDependency for one-time icon generation only; zero impact on production bundle

---

## Testing

- Open app in Chrome → three-dot menu → "Install CragWeather" should appear
- On iOS Safari → Share → "Add to Home Screen" should work
- Launching from home screen should open in standalone mode (no browser chrome)
- Verify no regressions: weather data still loads, API routes still respond normally
